# dystoretools — 客户交付计划

> 版本：v2 · 日期：2026-05-19
> 项目代号：抖店运营数字化助手（dystoretools）
> 交付方式：单店铺、单机部署，所有数据留在客户本机

---

## 一、项目目标（业务视角）

把店主每天在抖店后台需要手动盯的数据、要拍脑袋判断的异常情况、要花时间写的运营文案，全部沉淀成一台机器上自动跑的工具。开机即看，关键变化主动弹窗，文案与脚本一键生成。

工具部署在客户自己的电脑/服务器上，**数据不上云、不离开客户网络**。

---

## 二、最终交付内容（客户实际拿到的能力）

### 1. 抖店后台数据看板
- **订单**：实时订单列表、各状态计数、销售额
- **商品**：商品列表、类目分组、商品健康度
- **库存**：库存明细、低库与呆滞库自动诊断
- **评论**：全量评论、差评未回复列表、差评聚类（找痛点）
- **售后**：售后单列表、18 维度计数（退款/退货/换货/补寄等）
- **会员与受众**：会员看板（聚合/日/直方图）、受众画像
- **店铺治理**：体验分、违规与申诉、商家保障

### 2. 智能预警中心
- 差评激增、差评聚类痛点
- 库存低位、滞销
- 售后异常激增
- 销量异常（同比/环比下滑）
- 同行价格变动、同行直播开播（V2 子集）
- **触达方式**：工具内桌面通知 + 告警中心列表，重要级别可邮件/微信外推（按需配置）

### 3. AI 内容工坊
- 标题候选生成（多个备选）
- 详情页文案段落
- 短视频脚本（30/60/90 秒）
- 直播话术（开场/讲品/互动/促单）
- 评论与咨询的回复草稿（客户自己复制粘贴使用）
- **写作时会自动参考**：店里近期评论里反复出现的卖点与痛点

### 4. 罗盘搜索运营分析
- 搜索流量核心数据与趋势
- 搜索运营诊断与优化建议（AI 二次解读）
- 行业词排名、店铺排名
- 店播视频数据

### 5. 同行监控（精简版）
- 同行价格巡检（可配置同行店铺清单）
- 同行直播开播追踪
- 同行热卖品识别

> 不在本期交付范围：见第七节"明确不做的事"。

## 三、技术栈与系统简介

我们采用业内主流、可长期维护的技术组合，避免使用冷门或被淘汰的方案。客户可自行委托任何具备 Python / 前端能力的工程师接手二次开发。

### 1. 服务端（后台）
- **Python 3.12 + FastAPI**：业内主流的高性能 Web 服务框架
- **MySQL 8**：业内使用最广泛的关系型数据库，便于备份、迁移、二次分析
- **Redis 7**：主流缓存与任务调度中间件
- **Playwright**：微软出品的浏览器自动化引擎；工具用它模拟真人在浏览器里登录与翻页，**避免触发抖店的接口风控**

### 2. 前端（网页端）
- **React 18 + TypeScript**：业内主流前端组合
- **Ant Design Pro**：阿里出品的 B 端管理后台 UI 框架，操作风格与企业管理软件一致
- **ECharts**：百度出品的图表库，所有趋势图、饼图、漏斗图都用它
- **WebSocket 实时通信**：数据变化、预警触发不用刷新页面，秒级推送到屏幕

### 3. AI 大模型
- **主用：DeepSeek V4 Pro**（深度求索，国产顶级大模型，中文能力强，1M 超长上下文，适合一次性吃下整店评论）
- **备用：Kimi**（月之暗面）——主用模型不可用时自动切换，不影响业务连续性
- **隐私保护**：所有送入大模型的客户姓名、电话、地址、订单号**自动替换为占位符**再发出，不会向云端模型暴露真实客户信息

### 4. 部署与运维
- **Docker Compose 一键启动**：数据库、缓存、采集服务、网页端 4 个服务容器化运行，客户启停只需一条命令
- **单机部署**：所有数据存放在客户本机，不依赖任何云端服务（除调用大模型 API 外）
- **自动备份**：数据库每日定时备份到本地指定目录，可一键恢复
- **数据保留**：业务数据默认保留 1 年，超过 1 年自动归档；客户可随时导出全量 SQL 备份

### 5. 系统结构（简化示意）

```
                    [ 客户电脑 / 客户服务器 ]
                              │
   ┌──────────────────────────┼──────────────────────────┐
   │                          │                          │
 网页端                     采集服务                    数据库
 (浏览器)                (Playwright)                 (MySQL)
   │                          │
   │                          ├── 抖店后台
   │                          └── 同行店铺公开页
   │
   └── 大模型云端 API（DeepSeek / Kimi）
       仅传送脱敏后的运营数据
```

### 各节点明细

**节点 A（05-19 ~ 05-22）：接通真实数据**
- 工具部署到客户电脑（或客户提供的服务器）
- 客户配合在工具中扫码/登录一次自家抖店账号
- 工具首次抓取订单、商品、库存、评论、售后、会员看板数据
- 客户当面或线上评审一次：数据是否与抖店后台对得上

**节点 B（05-22 ~ 05-26）：业务面板与预警**
- 6 大数据面板全部跑通，关键指标与抖店后台一致
- 9 个定时窗口开始按生意节奏自动跑
- 预警引擎上线：差评激增、库存低位、销量异动、售后激增
- 桌面通知打通

**节点 C（05-26 ~ 05-29）：AI 与 V2 模块**
- 文案工坊全功能上线（标题/详情/脚本/话术/回复草稿）

- 罗盘搜索运营核心数据、诊断与优化建议
- 行业词排名、店铺排名、店播视频数据

**节点 D（05-29 ~ 06-02）：培训与验收**
- 现场或远程培训 1 次（≤2 小时）
- 用户手册、维护手册定稿交付

- 双方签收，进入 30 天质保期

## 七、售后服务

### 1. 质保期（验收后 30 天免费）

**覆盖范围**
- 功能性缺陷修复（已交付能力不能正常使用）
- 数据准确性问题（关键指标与抖店后台显著偏差）
- 抖店平台改版导致的字段错位、采集中断、接口路径变更
- 抖店风控策略调整导致的登录方式或采集节奏适配
- 大模型 API 故障切换问题

**响应与解决时效**

| 等级 | 定义 | 响应时效 | 解决时效 |
|---|---|---|---|
| **紧急（P0）** | 工具无法启动 / 数据库不可读写 / 全部面板不可用 | 4 小时内 | 24 小时内 |
| **严重（P1）** | 单一模块不可用，其他模块不受影响 | 24 小时内 | 3 个工作日内 |
| **一般（P2）** | 体验问题、文案错误、轻微显示异常 | 3 个工作日内 | 下次例行更新 |

**修复方式**：远程协助为主（TeamViewer / ToDesk / SSH 任选其一）；必要时上门支持（同城免费，异地差旅由客户承担）。

### 2. 质保期后服务（可选，按需选择）

| 模式 | 内容 | 计费方式 |
|---|---|---|
| **按次报修** | 单次故障远程支持 + 当次修复 | 按工时另议 |
| **年度运维包** | 12 个月不限次远程支持 + 抖店平台改版适配 + 季度健康巡检 + 大模型替换协助 | 包年价另议 |
| **不续费** | 客户继续自行使用，出问题再临时购买单次支持 | 仅按发生计费 |

### 3. 客户自助能力（一并交付）

为降低客户对售后的依赖，我们随软件交付：
- **维护手册**：日常巡检步骤、备份与恢复、常见报错处理
- **数据导出脚本**：一键打包 MySQL 备份与配置文件
- **紧急停机/重启脚本**：客户可自行执行
- **大模型 API Key 替换流程**：DeepSeek / Kimi 任一方涨价或停服时，客户自行更换密钥即可

### 4. 不在售后范围

| 不覆盖 | 说明 |
|---|---|
| 客户硬件故障 | 电脑、服务器、网络、磁盘损坏属于硬件维保范畴 |
| 客户自行修改代码或数据库导致的问题 | 改动前请联系我方评估 |
| 抖店账号本身被封禁、停业、降权 | 属于客户与平台之间的运营合规问题 |
| V3 范围的新增功能 | 千川、云图、多店铺等属于新需求，另行评估报价 |
| 第三方服务变更 | DeepSeek/Kimi 停服或大幅涨价属于上游变化，我方协助迁移但不承担费用 |

## 八、明确不做的事（避免后续误会）

| 不做 | 原因 |
|---|---|
| 自动回复飞鸽客服消息 | AI 只写草稿，由人复制粘贴；不接管账号说话，避免封号风险 |
| 多店铺管理 | 本期为单店铺单机版 |
| 千川（投流后台）数据接入 | 登录隔离 + 风控更严，纳入下期评估 |
| 云图（人群）数据接入 | 同上 |
| SaaS 云端托管 | 本期为本地部署，数据不出网 |
| 店铺装修 | 非数据/分析能力，不在本期范围 |
| 替客户运营店铺 | 工具是助手，不替代人 |

## 九、合作约定

### 客户需配合的事
1. **节点 A（首 3 天内）**：提供一台可联网的电脑或服务器（基础配置即可）
2. **节点 A（首 3 天内）**：客户本人在工具里完成一次抖店账号登录（扫码或输入，工具只读取浏览器会话，**不接触账号密码**）
3. **节点 D（最后一周）**：安排 2 小时培训时间，参与验收
4. **质保期内**：故障发生后及时提供必要的日志或截图，配合远程排查

### 我方承诺
- 工具运行在客户本机/本地，所有业务数据不出客户网络（仅大模型调用会发送脱敏后的内容到 DeepSeek/Kimi）
- 完整源码交付，客户拥有产权，可自行二次开发或更换服务商
- 验收期内 + 30 天质保期内的功能性缺陷免费修复
- 售后响应严格按第七节 SLA 执行

### 风险提示
- 抖店后台页面或接口若发生重大改版，质保期内免费适配；超出质保期按售后包内容处理
- 客户账号登录态若因平台风控失效，工具会主动提示客户重新登录一次（无需我方介入）
- DeepSeek / Kimi 等上游大模型服务的可用性、计费政策由各自服务商决定，不在我方可控范围
